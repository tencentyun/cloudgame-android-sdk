      • access

         abstract boolean access(List<String> instanceIds, boolean isGroupControl)

        Start the session and establish a connection to the cloud phone.

        After the local session object is successfully initialized (triggered by the onEvent with STATE_INITED), call this method to establish a connection to the specified cloud phone instance. Only after the connection is established can this session object interact with the cloud phone.

        Note:

        • Before calling this method, ensure:
          • The SDK has been initialized via init and the callback is successful.
          • The session object has been created via createTcrSession.
          • The session initialization success event (STATE_INITED) has been received.
          • A valid accessToken has been set (via init parameters or setAccessToken).
        • The connection result will be asynchronously returned via onEvent:
          • Success event: STATE_CONNECTED
          • Failure event: STATE_CLOSED
        Parameters:
        instanceIds - List of cloud phone instance IDs to connect to.
        isGroupControl - Whether to enable group control mode.
        Returns:

        Whether the connection request was successfully initiated (Note: The actual connection result is returned asynchronously).

        Returns true: The connection request was successfully initiated.

        Returns false: The request failed to send (usually due to invalid parameters or abnormal state).

      • changeSeat

         abstract void )>changeSeat(String userId, Role targetRole, int targetPlayerIndex, AsyncCallback<Void> callback)

        Switch the role and seat of a user (`userID`) to `targetRole` and `targetPlayerIndex` respectively. If the change succeeds, `onSuccess` will be called back. If there is another player on the target seat, the call will fail, and `MULTI_PLAYER_ASSIGN_SEAT_FAILED` will be passed in to `code` of the `onFailure` callback. Only the room owner can call this API. If it is called by a user who is not the room owner, the call will fail, and `MULTI_PLAYER_NO_AUTHORIZED` will be passed in to `code` of the `onFailure` callback. If the API call times out, the call will fail, and `ERR_TIMEOUT` will be passed in to `code` of the `onFailure` callback.

        Parameters:
        userId - The target user ID
        targetRole - The target role `Role`
        targetPlayerIndex - The target seat.
        callback - The callback for the setting result.

      • requestChangeSeat

         abstract void )>requestChangeSeat(String userId, Role targetRole, int targetPlayerIndex, AsyncCallback<Void> callback)

        Apply to the room owner to switch the role and seat of a player (`userID`) to `targetRole` and `targetPlayerIndex` respectively. If the change succeeds, `onSuccess` will be called back. This API can only be called by a user who is not the room owner. If it is called by the room owner, the call will fail, and `MULTI_PLAYER_IGNORED_HOST_SUBMIT` will be passed in to `code` of the `onFailure` callback. If the API call times out, the call will fail, and `ERR_TIMEOUT` will be passed in to `code` of the `onFailure` callback.

        Parameters:
        userId - The target player ID
        targetRole - The target role `Role`
        targetPlayerIndex - The target seat.
        callback - The callback for the setting result.

      • setMicMute

         abstract void )>setMicMute(String userID, int micStatus, AsyncCallback<Void> callback)

        Change the mic status of the `userID` user. The room owner can change the mic status of all users, but other users can change only their own mic status. If the room owner disables the mic of a user, the user cannot enable it. If the change succeeds, `onSuccess` will be called back. If a non-room owner user passes in `userID` of another user, `onFailure` will be called back, and MULTI_PLAYER_NO_AUTHORIZED will be passed in to ` code`. If `userID` is incorrect, `onFailure` will be called back, and `MULTI_PLAYER_NO_SUCH_USER` will be passed in to `code`.

        Parameters:
        micStatus - The mic status.
        callback - The callback for the setting result.
